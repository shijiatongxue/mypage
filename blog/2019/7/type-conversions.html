类型转换

作者：石嘉

日期：2019年7月23日

参考：JavaScript.Info

转换场景

ToString——输出内容时发生转换，或通过String(value)进行显示转换；

ToNumber——进行算数操作时发生转换，或通过Number(value)进行显示转换；

ToNumber转换遵循如下规则：

  • undefined => NaN
  • null => 0
  • true/false => 1 / 0
  • string => 字符串“按原样读取”，两端的空格被忽略。空白字符串变成0。出错变成NaN。

ToBoolean——进行逻辑操作时发生转换，或通过Boolean(value)进行显示转换；

ToBoolean遵循以下规则：

  • 0, null, undefined, NaN, "" => false
  • 其他值 => true

容易记错的点

ToNumber转换中，undefined => NaN, 而非0；

ToBoolean转换中，"0"或" " => true，因为只有为空的字符串才会转换为false。
